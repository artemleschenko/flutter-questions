# Что такое Generics? Расскажи про концепцию Soundness в системе типов Dart.

> `Generics` — это механизм параметризации типов, позволяющий писать переиспользуемый код (классы, интерфейсы, методы), который может работать с разными типами данных без потери безопасности типов (type safety). Концепция Soundness (надежность/строгость) системы типов Dart гарантирует, что программа никогда не перейдет в состояние, в котором фактическое значение переменной в памяти (runtime type) противоречит типу, заявленному на этапе компиляции (static type).

## Разбор

### Реификация (Reification) обобщений

Ключевое отличие Dart от многих других языков (например, Java с её "стиранием типов" — type erasure) заключается в том, что дженерики в Dart реифицированы. Это означает, что информация о параметрах типа сохраняется во время выполнения (run-time). Объект `List<int>`физически знает, что он содержит только целые числа, и проверка `myList is List<int>` отработает корректно.

### Soundness в Dart

Soundness работает на двух уровнях: статический анализ (во время написания кода/компиляции) и проверки времени выполнения (run-time checks). Компилятор блокирует большинство ошибок типизации на этапе статического анализа. Однако, для обеспечения гибкости (особенно из-за ковариантности), Dart генерирует скрытые проверки в машинном коде для отлова нарушений типов в рантайме.

### Ковариантность (Covariance) и потенциальные конфликты
Дженерики в Dart по умолчанию ковариантны. Это значит, что если `int` является подтипом `num`, то и `List<int>` признается подтипом `List<num>`. Ковариантность идеальна для чтения данных, но создает угрозу для Soundness при записи.

Поскольку Dart является "sound" языком, он не позволит нарушить целостность памяти при ковариантном присваивании, предотвращая "загрязнение" типов с помощью выброса исключения TypeError в рантайме:

```dart
void genericsDemonstration() {
  // 1. Реификация: тип известен в рантайме
  final list = <int>[1, 2, 3];
  print(list.runtimeType); // Вывод: List<int>
  
  // 2. Ковариантность (Covariance)
  // List<int> успешно присваивается переменной типа List<num>
  List<num> numbers = list; 
  
  // Чтение безопасно
  num firstNumber = numbers[0]; 
  
  // 3. Защита Soundness в рантайме
  // Статический анализатор пропустит это (double является подтипом num).
  // Но в рантайме это упадет с TypeError, так как под капотом numbers 
  // ссылается на память, выделенную строго под List<int>.
  // numbers.add(4.5); // Unhandled exception: type 'double' is not a subtype of type 'int'
}

// Ограничение дженериков (Bounded Generics)
// T обязан быть наследником Widget, что позволяет вызывать у него методы базового класса.
abstract class UIBuilder<T extends Widget> {
  T build();
}
```

### Практическое применение

В production-разработке Generics повсеместно используются для проектирования чистой архитектуры и обработки сетевых запросов. Самый частый юзкейс — создание классов-оберток (wrappers) для возврата данных из репозитория. Вместо того чтобы кидать исключения при ошибках API, разработчики создают алгебраические типы данных (на базе паттерна `Result` или `Either` из пакета `fpdart`), что позволяет статическому анализатору принуждать разработчика обрабатывать как успешный ответ, так и ошибку.

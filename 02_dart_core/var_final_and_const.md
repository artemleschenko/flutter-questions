# В чем разница между var, final и const? В каких случаях ты выберешь каждый из них?

> `var` используется для объявления изменяемой переменной с автоматическим выводом типа (type inference) на этапе компиляции. `final` определяет переменную, которая инициализируется во время выполнения (run-time) ровно один раз; ссылка на объект становится неизменяемой. `const` создает глубоко неизменяемую (deeply immutable) константу на этапе компиляции (compile-time), которая подвергается канонизации в памяти для оптимизации ресурсов.

## Разбор

Различия между этими ключевыми словами лежат в механизмах управления памятью, времени вычисления выражений и глубине неизменяемости.

- `var`

    При использовании var анализатор Dart выводит тип переменной на основе присвоенного значения. Важно: тип фиксируется во время компиляции. В отличие от dynamic, вы не можете позже присвоить переменной var значение другого типа.

- `final`

    Значение вычисляется в момент первого обращения или инициализации в рантайме. Ссылка final жестко привязана к объекту, но сам объект может быть изменяемым (shallow immutability). Например, если у вас есть final List<int>, вы не можете присвоить этой переменной новый список, но можете добавлять в него элементы через .add(). В контексте ООП поля класса, помеченные как final, должны быть инициализированы до завершения работы конструктора.

- `const`

    Значение должно быть известно компилятору до запуска приложения. const делает состояние транзитивно (глубоко) неизменяемым: в const List<int> нельзя добавить новые элементы во время выполнения (это вызовет исключение). Главная особенность под капотом — канонизация (Canonicalization). Dart собирает все идентичные const объекты и выделяет под них только одну область памяти.

```dart
void memoryDemonstration() {
  // final: Объекты создаются в рантайме. Разные адреса в памяти.
  final listA = [1, 2];
  final listB = [1, 2];
  print(identical(listA, listB)); // Вывод: false

  // const: Значения вычисляются при компиляции. Канонизация в действии.
  const listC = [1, 2];
  const listD = [1, 2];
  print(identical(listC, listD)); // Вывод: true (Один и тот же адрес в памяти)

  // Mutability check
  listA.add(3); // Успешно, сам объект мутабелен.
  // listC.add(3); // Runtime error: Unsupported operation: Cannot add to an unmodifiable list
}
```

### Практическое применение

В production-разработке на Flutter правило выбора строгое и иерархичное:

Используйте const везде, где это возможно. Особенно при создании виджетов (например, `const SizedBox(...)` или` const Padding(...)`). Фреймворк Flutter знает, что `const` виджеты никогда не меняют свою конфигурацию. Во время фазы `rebuild`, если фреймворк встречает const виджет, он полностью пропускает его обновление (и обновление его поддерева), что кардинально снижает нагрузку на CPU и Garbage Collector. Линтер `prefer_const_constructors` обязателен в серьезных проектах.

Используйте `final` для всего остального. Это стандарт де-факто для DI (внедренных зависимостей), полей классов-моделей и состояний. Поля в `StatelessWidget` обязаны быть `final`, так как сам виджет по контракту неизменяем (immutable).

Используйте `var` исключительно для локальных переменных внутри методов и функций, где жизненный цикл переменной очень короткий, тип очевиден (например, `var filteredUsers = users.where(...);`), и вам необходимо переназначать значение (например, счетчики циклов).

# Расскажи как реализовать Accessibility во Flutter приложениях

> Доступность (Accessibility / a11y) во Flutter реализуется через встроенный виджет **`Semantics`** (и его производные `MergeSemantics`, `ExcludeSemantics`), который описывает элементы UI для скринридеров (TalkBack на Android, VoiceOver на iOS). Также необходимо поддерживать динамическое масштабирование текста (`textScaler`), обеспечивать достаточный цветовой контраст и предусматривать навигацию с клавиатуры (`FocusNode`).

## Разбор

### Что такое Semantics?

Flutter рисует интерфейс с нуля на холсте (Canvas), поэтому операционная система по умолчанию не знает, где находятся кнопки, изображения или текст. Чтобы ОС могла озвучивать интерфейс для людей с нарушениями зрения, Flutter параллельно с деревом виджетов строит **Semantics Tree** (Дерево семантики).

Большинство стандартных виджетов Material и Cupertino (кнопки, тексты, чекбоксы) уже оборачивают свой контент в `Semantics` под капотом. Но для полностью кастомных виджетов (например, кастомный слайдер или переключатель) это нужно делать вручную.

*Пример кода*

```dart
Semantics(
  label: 'Кнопка добавления в избранное', // Что прочитает скринридер
  button: true, // Сообщает ОС, что это кликабельный элемент
  enabled: true,
  onTapHint: 'Добавить товар в список желаемого',
  child: GestureDetector(
    onTap: () {},
    child: const Icon(Icons.favorite), // Визуальная часть, непонятная для ОС
  ),
)
```

### Полезные виджеты для семантики

1. **`MergeSemantics`**: Группирует несколько семантических узлов в один. Полезно, если у вас есть строка с иконкой и текстом, и скринридер должен прочитать их как одну целую кнопку, а не как два разрозненных элемента.
2. **`ExcludeSemantics`**: Прячет поддерево от скринридера. Используется для декоративных изображений или фоновых анимаций, озвучивание которых только запутает пользователя.

### Масштабирование текста и контрастность

Люди со слабым зрением часто увеличивают размер шрифта в настройках телефона. Flutter поддерживает это "из коробки" через `MediaQuery`, но вы не должны ограничивать текстовые блоки жесткой высотой (`height`), иначе увеличенный текст обрежется (Overflow).

*   **Как получить масштаб:** Используйте `MediaQuery.textScalerOf(context)` (начиная с Flutter 3.16 это пришло на замену устаревшему `textScaleFactor`).
*   **Контрастность:** Текст должен иметь контраст к фону не менее 4.5:1 (для мелкого) и 3.0:1 (для крупного текста) согласно стандартам WCAG.

### Навигация с клавиатуры и D-Pad

Для пользователей с нарушениями моторики важна навигация кнопками (физической клавиатуры, переключателей или пульта ТВ).
Для этого используются виджеты **`Focus`** и класс **`FocusNode`**.
Каждый интерактивный элемент должен получать фокус и визуально подсвечиваться (например, через обертку `FocusableActionDetector`).

### Тестирование доступности

Во Flutter есть встроенный инструмент для визуальной отладки дерева семантики. Если включить его, весь UI покроется рамками с текстом, показывающими, что именно "видит" скринридер.

```dart
void main() {
  runApp(
    const MaterialApp(
      showSemanticsDebugger: true, // Включить отладку семантики
      home: MyApp(),
    ),
  );
}
```
